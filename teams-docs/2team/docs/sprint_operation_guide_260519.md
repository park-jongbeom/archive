## 3. 처방 5가지 (즉시 적용 — Week 3 D3 (수요일)부터)

### 3-1. Sprint Planning 정례화 (월요일 30~45분)

**왜 필요한가**: §2-3 데이터 — Sprint 1 → Sprint 2 전환 시 정식 Planning 회의 ❌. Mahnic (2014) 권고 = 학습자 팀 Sprint Planning 30~45분이 가장 효율적.

**실행 방법** (월요일 09:00~09:45):

| 시간 | 내용 | 산출물 |
|---|---|---|
| 0~10분 | **Sprint 목표 1줄 합의** (예: "AI 체시 + 콜렉션 플로우 데모 가능 상태") | 노션 1줄 |
| 10~25분 | **Backlog → Ready 이동** (이번 주 끝낼 수 있는 카드만 Ready로) | 보드 Ready 컬럼 |
| 25~40분 | **분담표 작성** (멤버별 카드 1줄씩 명시 — Issue # + 담당) | 노션 1표 |
| 40~45분 | **페어 매칭 결정** (§3-2) — 1주 1쌍 이상 | 노션 1줄 |

**핵심 원칙**:
- **이번 주 끝낼 수 있는 카드만 Ready로 이동** ([사내 가이드 §7-1](../../../final-project/docs/애자일/01_애자일_팀프로젝트_가이드.md) 보드 상태 규칙)
- Won't 항목 명시 ≥ 1건 ([예제 기반 FAQ §10 좋은 점부터](../../../ta-guides/애자일_예제기반_FAQ.md) "Won't가 없으면 범위 폭발" 안티패턴)
- 분담표는 회의 끝나는 즉시 노션 + 보드 동기화 (5/19 mom 미작성 패턴 회피)

**참고 양식**: [example2 1팀 03_backlog.md](../../../final-project/docs/애자일/example2/1팀/03_backlog.md) — Sprint 1/2 백로그 단위 작성자 분담표.

### 3-3. Daily Walking the Board (오른쪽 → 왼쪽 10분)

**왜 필요한가**: §2-4 데이터 — 5/19 mom 3명 분담 미작성 + [팀 규칙](./팀%20규칙) "09:00~09:50 정기 회의: 한 것 공유 + 할 것 공유 + PR & Merge" = 사람 보고 형태 + 사람 보고는 Patton/Gothelf의 **"yesterday/today" 안티패턴**.

**변경 권장** (현재 회의 시간 그대로, 형식만 변경):

**기존 양식 (yesterday/today)**:
- 박재민: 어제 한 일 (미작성 ⚠️) / 오늘 할 일 (미작성)
- 정원화: 어제 SCI 4건 / 오늘 (미작성)
- ...

**Walking the Board 양식 (Patton 권고)**:

```
[09:00~09:10] Walking the Board (10분)
- Done 컬럼 (오른쪽): 어제 머지된 카드 1줄씩 확인 — "DASH-006 머지 ✅"
- In review: 6/5 ⚠️ → 누가 오늘 18:00까지 리뷰? (페어 매칭 적용)
- In progress: 5명 카드 1장씩 — "오늘 끝낼 수 있는가?" (yes/no/막힘)
- Ready: 막힘 해소되면 진입 카드 누구 / 언제
- Backlog: (회의 외 처리)

[09:10~09:30] PR & Merge + Q&A (20분)
- (기존과 동일)
```

**핵심 질문 (Patton/Gothelf)**:
- "What have you **finished** since yesterday?" (yesterday 안티패턴 우회)
- "What will you **finish** by **when**?" (by when 명시 강제)
- "어디서 막혔거나 **느려졌나**?" ("blocker 없음" 안티패턴 우회)

→ **5분 정도 추가 효과**: 보드 카드 단위로 발화 → 분담 미작성 멤버 자동 호명 → 5/19 박재민/정원화/정재훈 분담 미작성 패턴 자동 해소.

**참조**: [외부 베스트 프랙티스 §1-2](../../../operation/docs/research/external_practices.md).

### 3-4. 페이스 점검 시스템 — R-burn 예방 (매일 PM 회의 끝 1분)

**왜 필요한가**: §2-1 데이터 — R-burn 2명 동시 패턴 + 18h 9 commits 중 5건이 새벽/야간 (55.6%) = R-burn 임계 도달.

**실행 방법** (PM 회의 17:25~17:30 마지막 1분):

```
[17:29] 신호등 1단어 회고 (각 30초)
- 박재민: 🔴/🟡/🟢 + 1단어 (예: "🟡 분담")
- 정원화: 🔴/🟡/🟢 + 1단어 (예: "🟡 페이스")
- 김명준: 🔴/🟡/🟢 + 1단어 (예: "🟢 진척")
- 김태환: 🔴/🟡/🟢 + 1단어 (예: "🟢 마감")
- 정재훈: 🔴/🟡/🟢 + 1단어 (예: "🟢 이해")

같은 색이면 align ✅
🔴 1명 이상이면 → 회의 후 강사 1:1 (10분)
🟡 2명 이상이면 → 내일 AM 첫 안건 자동 격상
```

**R-burn 자가 점검 룰** (각자 1주 1회 일요일 저녁 셀프 체크):

| 지표 | 🟢 | 🟡 | 🔴 |
|---|---|---|---|
| 1주 자정 이후 (00:00+) commit 수 | 0 | 1~2건 | 3건+ |
| 1주 새벽 (02:00~06:00) commit 수 | 0 | 1건 | 2건+ |
| open assign 카드 | ≤ 3 | 4 | 5+ |
| 본인 "힘들다/무리한다" 발언 | 없음 | 1회 | 2회+ |

→ 🟡 1개 이상 + 🔴 1개 이상 = **다음 월요일 Sprint Planning에서 분담 조정 합의**.

**Sprint Planning에서 R-burn 시 처방**:
1. **작업 분산** (페어 매칭 강제 — §3-2)
2. **카드 압축** (이번 주 카드 1~2건 다음 주 이월)
3. **새벽 작업 금지 합의 (자율)** — 22:00 이후 commit X 권장

**참조**: [캡스톤 v2.3 §III Burnout 대응](../../../ta-guides/파이널프로젝트_캡스톤_심화_레퍼런스_2026-05-14.md) + [risk_taxonomy.md R-burn](../../.shared/risk_taxonomy.md).

### 3-5. Sprint Retro + 다음 Sprint Planning (금요일 30~40분)

**왜 필요한가**: §2-3 데이터 — Sprint 1 → Sprint 2 전환 시 Retro 회의 ❌. [example2 1팀 07_retrospective.md](../../../final-project/docs/애자일/example2/1팀/07_retrospective.md) 양식 권장.

**실행 방법** (금요일 17:00~17:40):

| 시간 | 내용 | 양식 |
|---|---|---|
| 0~10분 | **이번 Sprint Done 카드 1줄씩 회고** | Walking the Board (오른쪽만) |
| 10~25분 | **KPT 또는 Start/Stop/Continue** (각자 포스트잇 3장 = 15장) | KPT 권장 ([예제2 1팀 07_retrospective.md](../../../final-project/docs/애자일/example2/1팀/07_retrospective.md)) |
| 25~35분 | **Action item 합의 ≤ 3건** (다음 Sprint 적용) | 노션 |
| 35~40분 | **다음 Sprint 목표 1줄 + 분담 1표** (§3-1 일부 미리) | 노션 |

**KPT 양식 (한국 부트캠프 표준)**:
- **Keep**: 이번 Sprint에 잘된 것 (다음 Sprint 유지)
- **Problem**: 이번 Sprint 문제 (원인 명시)
- **Try**: 다음 Sprint 시도할 것 (Action item)

**핵심 원칙**:
- **인물 비판 X / 코드·프로세스 비판 ✅** ([예제 기반 FAQ §10](../../../ta-guides/애자일_예제기반_FAQ.md))
- "정원화가 새벽 작업 한다" ❌ / "팀 차원에서 새벽 작업 자제 합의" ✅
- **Action item 3건 초과 X** — 다음 Sprint 적용 가능한 수만

**참조**: [외부 베스트 프랙티스 §4-2 Retrospective 양식](../../../operation/docs/research/external_practices.md).

## 6. Q&A 예상

### Q1. "Sprint Planning 30~45분이 길지 않나?"
A. 데일리 50분(09:00~09:50)을 30분(09:00~09:30)으로 줄이고, **월요일만 45분으로 늘리는 형식**이 [Mahnic (2014)](../../../operation/docs/research/external_practices.md) 권고. 1주 총 시간은 동일 또는 -10분 단축.

### Q2. "신호등 회고가 형식적이 되지 않을까?"
A. 형식적이 되는 것을 막는 룰: **🔴 1명 = 강사 1:1 자동 트리거**. 🔴 표시 = 강사 케어 보장 = 학생 입장에서 표시 인센티브 ↑. [외부 베스트 프랙티스 §4-2](../../../operation/docs/research/external_practices.md).

### Q3. "페어 매칭이 시간 비용이 너무 크지 않나?"
A. **1주 1쌍 = 1쌍이 1주 동안 같이 작업하는 게 아니라, 1주에 1회 (예: 화요일 11:30 코드 리뷰 시간)에 같이 PR 리뷰**. 부담 ↓ + R-burn 분산 ↑.

### Q4. "5/26 목표 8건 다 달성해야 하나?"
A. **5건 이상 = 정착 sentinel**. 8건 모두 = 다른 팀(1팀/3팀) 베이스라인으로 인용 가능한 모범 sentinel. 5건 미달 = Week 4 (5/27~) 시작 시 강사 1:1 추가 케어.
