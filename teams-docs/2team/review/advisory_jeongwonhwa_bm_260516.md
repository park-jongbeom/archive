> **검토 시점**: 2026-05-16
> **검토자**: 보조강사

## 🎯 면담 조언 핵심 (Top Priority)

### ① 동기 인정 먼저 — "BM이 본인 창업 아이템인 거 알고 있어요"
- 방어 해제용. 정원화의 완성도 욕심을 *부정*하지 말고 *맥락을 이해한 상태에서 같이 정리*하는 톤.
- 1~2분 내 짧게. 이걸 빼면 이후 조언 전부 방어적으로 수용됨.

### ② "다음 주차 sprint에 필요한 문서인지 우선순위 확인"
- 애자일 원칙 **JIT(Just-In-Time) 문서화** 와 정확히 정합.
- 강사 5/14 PM 발화 *"백로그 그렇게 잘 쪼개야 되는 거예요 안 그러면 여기 치여서 진짜 잠 못 자요"* — **강사 권위로 백업되는 조언**. 면담에서 직접 인용 권고.
- 본인 입으로 답하게 유도 질문: **"이번 sprint에서 팀이 의사결정하려면 BM 중 어떤 부분이 필요할까요?"**

### ③ "필요한 문서만 빠르게" — 단, '빠르게'의 정의 합의
- 정원화 입장에서 '빠르게' = **창업 자료니까 완성도 95% 빠르게** 일 가능성.
- 사용자 의도 = **80% 수준 초안 빠르게**.
- 어디까지가 "이번 주 충분한 수준"인지 본인이 직접 정의하게 유도. 보조강사가 정해주면 본인 욕심 통제 안 됨.

### ④ ⭐ 다이어그램 / 순서도 시각화 병행 (가장 강한 한 수)
- **이유 3가지**:
  1. 애자일 팀은 긴 텍스트 BM 안 읽음. 김명준/박재민/정재훈 모두 commit 중심 — 시각 1장이 텍스트 5장보다 팀 이해 효율 압도적.
  2. 정원화 본인 창업 자료로도 시각화 = 투자자/협업자 설득 무기. 부트캠프 산출물이 그대로 창업 자료로 살아남음.
  3. 시각화 작업 자체가 사고 정리 — 글로 다 못 쓴 빈틈을 다이어그램이 강제로 드러냄.
- **도구 추천 (2단계)**:
  - **1차 (이번 주)**: Mermaid — 가장 빠름, git 친화, 팀원이 PR로 수정 가능. 애자일 iteration 최적.
  - **2차 (다음 sprint)**: Figma — 창업 자료로 그대로 살아남는 퀄리티. (figma MCP 활용 가능)
  - 보조: Whimsical / Excalidraw — user flow 빠른 스케치용

### ⑤ ⭐⭐ 부트캠프 BM ↔ 창업 BM 분리 (해방감을 주는 핵심 메시지)
- **현재 상태 추정**: 한 개 BM 문서에 *부트캠프 팀 공유 + 본인 창업 자료* 두 목적을 다 담으려고 함 → 완성도 욕심 + R-burn 직결.
- **분리 제안**:
  - **부트캠프용 (이번 주 sprint)**: 시각화 1장 + 한 페이지 요약 = 팀 공유 최소 단위
  - **창업용 (부트캠프 종료 후 본인 별도 작업)**: 정원화 개인 자산으로 보관 + 부트캠프 끝나고 본인 페이스로 보강
- **효과**: *"창업 자료를 포기하라"* 가 아니라 *"부트캠프 timebox 안에서는 가볍게, 진짜 창업 자료는 부트캠프 후 본인 페이스로"* — 동기 보호 + R-burn 차단 동시 달성.
- **손실감 차단**: 부트캠프용 시각화가 그대로 창업 자료 base가 되도록 **재사용 가능한 도구**(Figma) 로 만들면 정원화에게 손실감 없음.

### ⑥ 팀원 공유 = 일방향 X → 양방향 워크 세션 ⭕
- 단순 공유 = 정원화가 다 쓰고 팀원이 수동 수용 → 본인 burnout + 팀 이해도 안 올라감.
- 권고: **15분 짧은 워크 세션** 형태. 시각화 1장 띄우고 팀원이 즉시 질문 / 수정 제안.
- 참고 모델: 5/15 AM 3팀 친구 추가 UID 흐름 자생적 토론 패턴 — 2팀에도 이식 가능.

### 정원화 R-burn 이력 (강사 직접 발화)

**5/14 PM**:
- *"백로그 그렇게 잘 쪼개야 되는 거예요 안 그러면은 여기 치여서 진짜 잠 못 자요"*

**5/15 AM**:
- *"다음 번 차스에서는 이슈 단위를 좀 낮춰보세요 이슈 하나하나의 단위를 좀 더 가볍게... 테스트를 해 보셔가지고 팀에 맞는 스타일로 그리고 팀원들의 그 역량에 따라서 배분"*

→ 강사가 이미 2일 연속 직접 케어 발화. **사용자 의견 "필요한 문서만 빠르게"가 강사 권유와 정확히 정합**. 면담에서 강사 발화 인용으로 권위 부여 가능.

→ 본인 창업 아이템이라는 강한 동기가 R-burn을 **더 위험하게** 만듦 (완성도 욕심이 부트캠프 평균보다 강함).

### 애자일 정합성 (2팀 방법론)

- 애자일 원칙: *Working software > comprehensive documentation*
- BM 문서도 **이번 sprint 의사결정에 필요한 부분만** 작성하는 것이 정석
- iteration마다 BM 한 페이지 업데이트 = 정원화에게 부담 적은 접근

### 2팀 현황 (5/15 AM 기준)

- v2 양식 사전 통지 이행률 **50%** (3팀 중 최고 ⭐)
- 단, **신호등 회고는 3일 연속 미시행** → 1:1 면담에서 정착 시도 가능
- R-burn-2 김명준 신규 발현 (5/14 새벽 1시 작업) — 2팀 burn-out 패턴 광역화 우려
- 정재훈 R-out-2 회복 sentinel (자기성찰 발화 본격화)

| **Figma** | 창업 자료로 그대로 사용, MCP 활용 가능 | 학습 비용 | 다음 sprint 정식화 ⭐ |
