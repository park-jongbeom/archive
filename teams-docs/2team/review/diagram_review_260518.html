🚨 Critical (구현 전 합의 권장)

1. d1 온보딩 — pending sync 회수 경로 부재

LS-007 7개 Summary 초기 생성 → Local 1차 저장 ✅ → Firestore 배치 저장 라인은 있지만, Firestore 저장 실패 시 pending sync로 빠지는 회수 경로가 표시되지 않음.

다른 모든 다이어그램(Correction d5, SRS d6, Statistics d7/d8)은 Rollback + Retry + pending=dirty flag 패턴을 일관되게 그렸는데, 가장 critical한 초기화 단계에만 누락 = 일관성 깨짐.

💡 권장: d1에도 Firestore 실패 → local만 유지 + pending sync 등록 분기 추가. 초기화 실패 시 앱이 깨지면 가장 큰 사고.

2. d4 AI Chat 세션 — reconnect / restore 경로 미시각화

문서 목적 명세에 "reconnect 흐름" + "re-entry restore" 포함이 적혀있는데, 실제 다이어그램은 Bootstrap → PTT → stream → final turn save → Session Memory append 단방향 happy path만 그려짐.

step 10 "CHAT-006 종료 및 재진입 후 누적 turn 표시" 박스가 있긴 하나 LiveSession 끊김 → 재연결 → activeSessionId 복구 분기가 빠짐.

💡 권장: Gemini Live 연결 끊김 / 네트워크 단절 시 partial transcript 처리 (폐기 vs 임시저장) + activeSessionId 기준 재진입 분기 추가.

⚠️ 검토 권장

3. d5 Correction — recentFullContext 빈 경우 처리 불명확

"후보 추출 결과 = Empty" 분기는 있는데, 그 위에 있는 recentFullContext 조회 → Session Memory 조회 단계 자체의 실패 / 빈 결과 분기가 모호.

AI Chat 신규 사용자(첫 세션, 메모리 비어있음)가 Correction 진입하면 어떻게 동작하는지 화면에서 알 수 없음.

💡 권장: recentFullContext = null/empty 분기를 명시 (Correction 화면이 "교정할 발화가 아직 없습니다" 비어있는 상태로 보이는지).

4. d6 SRS — Again / Hard / Good / Easy 4분기 머지 흐름 단순화 필요

ReviewSchedulePolicy 박스 안에 4개 결과 텍스트가 한 박스에 묶여 있어, 카드별 next_review_at 계산이 각 평가별로 어떻게 다른지가 시각화 안 됨.

SM-2 핵심이 "평가별 다른 다음 review 일정"인데, 다이어그램에서는 그 핵심이 안 보임.

💡 권장: ReviewDecision 박스를 4갈래 mini-flow로 펼치거나, 별도 표 / sub-diagram으로 분리.

5. d1-B 저장 책임 매트릭스 — 정합성 잘못된 셀 의심 2건

  • SessionSummary 행 / Firebase 칸: "preload cache"라 적혀있는데, 다른 Summary(Dash, Flashcard)는 모두 "sync"로 표기됨. preload는 Local 칸 책임이고 Firebase 칸은 sync여야 일관됨.
  • Flashcard 행 / Room 칸: "local first 저장"은 맞으나 Firebase 칸 표기가 흐려 확인 필요 (batch sync? background sync?). d5 Correction은 batch, d6 SRS는 background로 그려 두 다이어그램이 충돌 가능성.

💡 권장: 책임 매트릭스를 단일 source of truth로 두고, d5 / d6 sync 표기 문구를 매트릭스와 일치시키기.

6. 전 다이어그램 공통 — 권한 / 오프라인 시작 경로 누락

d1 온보딩에 "Google 로그인 → Auth / Error" 분기는 있으나, 앱 최초 실행 시 오프라인 상태에서 Firestore 접근 실패 케이스가 모든 다이어그램에서 빠져있음.

김명준이 5/17 겪었던 (default) DB 이름 사고처럼 환경 / 네트워크 문제는 다이어그램 표준 분기로 박아두면 디버깅 시간 단축.

💡 권장: 각 Entry 박스 옆에 "오프라인 시작 → 캐시 사용 또는 차단 화면" 1줄 추가.

💡 작은 표기 일관성 (cosmetic, but matters)

7. swim-lane 색상 통일

  • d4(AI Chat)만 swim-lane 헤더 색상이 노랑계열 단일 — 다른 7장은 4컬럼 색 분리. 통일하면 한눈에 비교 가능.
  • d3(Dashboard) 진입점 맵의 selectedLearningLanguage 미설정 → fallback success → Error 라인 굵기 / 방향이 모호 → fallback이 실패하면 어디로 가는지 추적 어려움.
