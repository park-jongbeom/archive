## ⚠ 사전 주의

1. 태환 본인이 도입에서 *"두 가지 고민이 있는 것 같아요"* 라고 안건 명확히 제시. 능동적 진입. **본인이 "막막한 상태는 아니다" 라고 자평**한 점도 함께 봐주실 것.
2. 본 정리는 보조강사 시각. 일부 인용은 음성 인식 오류 보정([대괄호] 표기).
3. **태환의 DM 후속 채널 자율 확보 발화** *("또 DM 할게요")* — 보조강사 라인 + DM 보조 라인 견고화 중. 강사진 직접 청취 시 이 채널을 깨지 않는 톤 권장.

## ⭐ 김태환이 표현한 어려움 5건 (중요도 순)

### 1. ⭐⭐⭐ 추상적 아이디어 → 구체화 격차 (대화 절반 차지, 본인 1번 안건)

**본인 발화 (요지)**:

> *"여기까지 나와 있는 상태라고 하면 그럼 그 무엇을 어떻게 할 것인가 난 질문을 해야 하잖아요. 제가 그걸 구현을 하려면 근데 그 어떻게 할 것인가에 대해서 너무 추상적이었다. 지금 플로우를 작성을 다 했지만 거기서도 많이 충분히 구[체]화가 안 되어 있는 상황이라고 생각을 하고요"*

- 아이디어 → 구현 이행 단계의 **구체화 격차** 본인 자각
- 본인 작성 플로우조차 충분히 구체화 안 되었다고 평가
- 비유 인용: *"고속도로 칸막이 위를 달리는 구조차를 개발해보자"* — 문과/이과 사고 차이 (현장 제약 무시한 채 추상 아이디어 내려오는 상황)

**중요한 부수 발화 — 정원화 의존 패턴 ⭐⭐⭐**:

> *"박원하님이 앞서서 처리[하면서] 계속 해 주시면서 구[체]화 되고 있어서 시간이 지나면서 해결된 문제라고 생각은 하세요. 지금 당장 계속 힘들다"*

> *"원하님이 진행하면서 [구체화]되고 그렇게 하고 그래서 나누기가 힘들어요"*

→ **정원화 R-burn 자율 추가의 구조적 원인을 태환 입에서 노출**. 정원화가 "앞서서 처리"하면서 구체화 → 다른 팀원이 받아갈 분담 단위 형성이 그때그때 → 분담 어려움 → 정원화 본인은 계속 앞서가야 함.

→ **강사진 활용 권장**: 정원화 케어 시 데이터 보강 자료로 활용. 단 **정원화에게 직접 인용 X** (태환의 1:1 비밀 보호).

### 2. ⭐⭐⭐ 학습 사이클 차단 (대화 1/3 차지, 본인 2번 안건)

**중간 프로젝트 vs 현재 비교**:

> *"[파이널] 중간 프로젝트를 했을 때는 AI 도움받아 가지고 초안을 작성하면 그거를 100% 분석을 할 시간이 있었어요"*

> *"[AI 부분만] 정확하게 제가 알고 넘기는 [느낌]이 그 [느낌]이에요. 그래서 그거 그렇게 쳐내고 약간 [의문]이 생겼을 때 아 그럼 내가 이제 저쪽 [코드]를 다시 한번 돌아봐야지 명확하게 다 알고 싶으니까 라고 돌아가려고 하면 또 다른 일이 내려오고 공부 좀 하고 올게요 그러면 아 이거 할 거 있다"*

→ **본인의 건전한 AI 활용 원칙**(이해 후 넘김)이 작동하지 못하는 환경 = 새 임무가 계속 쌓여 학습 마무리 시간이 잠식되는 패턴 ⚠️

→ **강사진 액션 권장**:
- 회의 중 *"공부 시간이 보장되고 있나요?"* 직접 청취 의제
- 본인의 *"공부 좀 하고 올게요"* 발화권 보장 합의 (스프린트 운영 룰 수준)

### 3. ⭐⭐ 본인의 능동적 떠맡기 — R-burn 위험 신호

**본인 자술**:

> *"내가 좀 할 수 있을 것 같으니까 해보자 이러신 것 같아요. 후자가 가깝기도 하고요"*

> (부담 표현 시도) *"조금 [어렵다] 약간 나한테 좀 많은 것 같다 라고 말을 했을 때..."*

→ 5/16 정원화 1:1 *"본인 능력에 강박적 align"* 패턴과 **동일 유형 ⚠️**. 단 태환은 **본인의 페이스 자율 케어 합의 발화도 보유**:

> *"지금 단기 프로젝트니까 1.2 1.3으로 잡고 그리고 어차피 시작할때 말했거든요. 저 하루 아니면 이틀 이 프로젝트 하면 휴가 쓸거다"*

→ **강사진 액션 권장**:
- **태환 휴가 시점 합의 의제 격상** (1~2일, 단기 프로젝트 평균치 기준). 본 1:1에서 본인이 자율 자술. PM 또는 다음 AM에서 회의 의제로 공식 마련 권장.
- 보조강사 가르침(일정 1.5배·MVP 우선·주말 일정 금지) 회의에서 재인용 권장 — 김명준 R-burn-2 재발과 동일 패턴 예방 차원에서 **2팀 공통 가르침**으로 격상

### 4. ⭐⭐ 멘토 역할 비용 (재훈 케어 회상)

**본인 자술**:

> *"재훈님이랑 같이 [대시]보드 쪽을 맡았었거든요. 근데 재훈님한테 오히려 제가 이거 이거 하면 되고 저거 하면 된다고 구[체]화를 엄청 해드렸어요. 근데 [후배]는 구[체]화를 더 해줘도 못하고 이런 [경우]도 있기도 하셔가지고 그래서 더 쉽지 않았던 것 같습니다. 근데 어떻게 되다 보니 그 [후배]는 없어졌어요"*

→ 5/18 PM **태환 멘토 역할 확립 sentinel ⭐⭐⭐** 의 **이면 = 멘토 비용 노출**. 5/18 11시 코드 리뷰(김명준 멘토) + 재훈 케어 + 본인 작업 = 3중 부담.

→ **강사진 액션 권장**:
- 회의 격려 시 멘토 역할 격려를 **본인 작업 격려와 분리** (멘토 비용 인지를 가시화하는 톤)
- 5/18 PM 격려 (멘토 역할 확립)는 유지하되, **부담 분산 합의를 동반** ("멘토 역할이 본인 작업 일정을 침범하지 않는지" 본인 입으로 확인)

### 5. ⭐ 2차 개발 기간 미합의 (운영 의제)

**본인 자술**:

> *"그거를 받아서 [면적]을 [확인]해보고 2차 개발 기간을 정하자 라고 남아있는 상태인데 그때 제가 뭐 사실 말을 안 꺼내도 되네요"*

→ **2차 개발 기간 = 미합의 상태**. 본 1:1 가르침(1.5배 / MVP / 휴가) 받은 후 본인이 직접 안건 제기할 의도였으나, 보조강사가 *"강사가 회의에서 직접 의제로 던질 것"* 으로 시사한 결과 *"제가 말 안 꺼내도 되네요"* 로 정리됨.

→ **강사진 액션 권장**:
- **다음 회차 강사 사전 통지 의제 1순위 자동 격상**: 2차 개발 기간 정의 안건
- 본인 자율 페이스 합의(1.2~1.3배) + 휴가 합의를 이 의제에 묶어서 일괄 처리 권장
