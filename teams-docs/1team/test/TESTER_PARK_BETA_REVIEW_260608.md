## 0. 1줄 결론 (디바이스 관점)

> **로컬 카페인 기록·수면 리포트 코어는 정상.** 그러나 README 간판(AI·푸시·워치)은 **버그가 아니라 "기능 자체가 없음/고장"** — 본인 단말에선 "고장처럼 보이지만 크래시는 안 나는" 경우가 대부분. **디바이스로 볼 진짜 크래시는 사실상 없음**(코어 견고). 따라서 **간판 기능 전멸인 워치 3종을 제일 먼저 친다.** (Health Connect 미설치 경로도 리포지토리가 `runCatching`으로 방어 → 크래시 아님, 빈 처리. 아래 D-01 정정 참고.)

## 1. 디바이스로 바로 보이는 결함 / 의심 지점

### 🟡 D-01. Health Connect 미설치 시 수면/리포트가 크래시 없이 빈 처리되는지 (가드 검증)
> ⚠️ **정정(6/08 코드 재확인)**: 당초 "유일한 실크래시 후보 / 최우선"으로 봤으나, 리포지토리가 HC 호출을 방어하고 있어 **크래시 아님**. [SleepRepositoryImpl.kt:66-74](../repo/app/src/main/java/com/snoffee/app/data/repository/SleepRepositoryImpl.kt#L66)가 `runCatching { healthDataSource… }.getOrDefault(emptyList())`로 감싸므로, HC 미설치 시 `getOrCreate()`가 던지는 `IllegalStateException`을 잡고 **빈 목록 반환** → Room 데이터와 머지. 온보딩도 [getSdkStatus 가드](../repo/app/src/main/java/com/snoffee/app/presentation/onboarding/permission/OnboardingPermissionScreen.kt#L233)로 스토어 유도. → **크래시가 아니라 "가드가 잘 도는지" 확인하는 항목.**
- **배경**: Health Connect는 **AOS14부터 OS 내장**. **Note10+(AOS12)엔 기본 미설치** → Play에서 별도 설치해야 함. 참조: [Health Connect 시작하기 — AOS13 이하는 Play 별도 설치](https://developer.android.com/health-and-fitness/health-connect/get-started)
- **동작**: Note10+에서 **Health Connect를 설치하지 않은 채** 수면 탭 / 리포트 진입.
- **기대(정상)**: 크래시 없이 **빈 상태/"데이터 없음"** + (온보딩) 설치 유도.
- **깨지면(이상)**: 만에 하나 진입 순간 **앱이 튕기면** 가드가 안 도는 경로가 있다는 뜻 → 즉시 보고 + 재현 단계 기록. (현 코드 분석상 크래시 가능성 낮음)
- 추가: HC 설치는 했지만 **READ_SLEEP 거부** → 수면 목록 빈/"0h 00m", 크래시 없는지도 확인.

### 🔴 D-02. 워치 — 항상 "연결 끊김"으로 뜨는지
- **동작**: 갤럭시 워치 페어링 후 Snoffee 워치 앱 진입.
- **기대**: "연결됨" + 데이터.
- **깨지면**: **페어링돼 있어도 무조건 "핸드폰 유실 또는 연결이 끊어졌습니다"** 표시. (워치 영역에서 가장 먼저 보이는 결함)

### 🔴 D-03. 워치 — 잔류 카페인 게이지가 항상 0 mg인지
- **동작**: 폰에서 카페인 기록 → 워치 홈 숫자 확인.
- **깨지면**: 폰에 데이터가 있어도 워치 mg가 **0으로 고정**. (위험도/대사시간 등 다른 값은 동기화될 수 있으나 **mg 숫자만** 0)

### 🔴 D-04. 워치 — 컷오프 알림이 아예 안 오는지
- **동작**: 폰에 카페인을 많이 기록하고 취침시간 근처까지 대기.
- **깨지면**: **워치 컷오프 알림이 영원히 안 옴**(스케줄 자체가 없음). 죽은 기능임을 확인.

### 🟠 D-05. 워치 설정 토글이 폰에 반영되는지
- **동작**: 워치에서 알림/수면 설정 토글.
- **깨지면**: **폰에 아무 반영 없음**.

### 🔴 D-06. AI 컷오프 / 주간 인사이트 — 화면에 버튼·기능이 존재하는지
- **동작**: 앱 전 화면에서 "AI 컷오프 추천", "주간 AI 인사이트" 기능을 찾아본다.
- **기대(README)**: Gemini 기반 개인화 추천.
- **깨지면**: **그런 기능/버튼이 화면에 아예 없음**(미연결). 홈에 보이는 컷오프 시간은 AI가 아니라 로컬 계산값. → "탭하면 크래시"가 아니라 "기능 부재"로 보고.

### 🔴 D-07. 푸시 알림(FCM) / 종 아이콘이 동작하는지
- **동작**: 앱바 **종모양 아이콘** 탭 + 푸시가 오는 상황(컷오프 등) 대기.
- **깨지면**: 종 아이콘 **무반응**, **푸시 자체가 안 옴**(푸시 기능 미구현). 죽은 기능 확인.

### 🟠 D-08. 비행기 모드 첫 실행 — 음료 검색이 빈 화면인지
- **동작**: 비행기 모드로 첫 실행 → 음료 추가/검색.
- **깨지면**: **에러 메시지 없이 음료 목록이 비어** 보임(원격 음료 데이터를 못 받음). 사용자가 "검색 결과 없음"으로 오해.

## 2. 🧭 앱 방향성 기반 — 일부러 시도할 디바이스 시나리오

카페인·시간·수면 앱 특성상 경계값을 직접 친다:

1. **자정 넘김** — 음료 기록 후 자정 지나며 관찰: "오늘 섭취량"이 **0으로 리셋**되고 **잔류 카페인은 이월**되는지. 게이지가 어색하게 튀지 않는지.
2. **0 카페인 커스텀 음료** — 카페인 0짜리 음료 등록 → 크래시 없이 "안전"으로 표시되는지.
3. **시계 뒤로 돌리기** — 기기 시간을 과거로 → 카페인 화면 진입 시 음수 시간/크래시 없이 처리되는지.
4. **여러 날 다회 기록** — 며칠에 걸쳐 5잔+ → 누적 잔류량·주간 리포트가 말이 되는지.
5. **수면 기록 0건** — HC는 있는데 수면 데이터가 없을 때 → 빈 상태 뷰가 뜨는지(크래시 X).
6. **앱 업데이트 후 데이터** — (가능하면) 버전 올린 빌드 재설치 시 **기존 로컬 카페인 기록이 사라지는지**(파괴적 마이그레이션) — 사라지면 기록.
7. **Z Fold4 폴드/언폴드 중 상태 유지** — 음료 입력·리포트 보는 중 **펼침↔접힘** → 입력값·스크롤·선택이 유지되는지(재구성으로 날아갈 수 있음). 참조: [Compose 폴드 인식 — 상태 보존](https://developer.android.com/develop/ui/compose/layouts/adaptive/foldables/make-your-app-fold-aware)
8. **권한 카드 문구 검증** — 온보딩 권한 안내에 "심박수/활동량 기반 반감기 계산" 류 문구가 있는데 **실제로 그 권한을 쓰거나 묻는지** — 안 쓰면서 문구만 있으면(오해 소지) 기록.
