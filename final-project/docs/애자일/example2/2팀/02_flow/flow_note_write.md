# User Flow — 쪽지 작성 (FLOW-NOTE-WRITE)

## Flow ID

- `FLOW-NOTE-WRITE`

## 목표

- 사용자가 새 쪽지를 작성해 저장하고, 지도/목록 맥락에서 확인 가능한 상태로 만든다.

## 시작 조건

- 로그인됨.
- 쪽지 작성 화면 진입 가능(`F-NOTE-01` 진입 경로는 팀 합의: FAB, 지도 롱프레스 등).

## 주요 단계

1. 작성 화면에서 텍스트 입력(명세상 필드 범위는 Issue AC로 고정).
2. (Should) 카메라/사진 첨부 `F-NOTE-08` — 권한·압축·실패 처리.
3. 저장 요청(Loading).
4. 성공 시 지도에 마커 반영 또는 목록 갱신(Success).
5. 실패 시 Error + Retry.

## 종료 조건

- 성공: 서버/BaaS에 저장되고 사용자에게 완료 피드백.
- 실패: 검증 오류·네트워크 오류 메시지.

## 예외/상태(최소)

- 빈 내용·길이 제한
- 이미지 용량·권한 거부
- 오프라인/timeout
- 중복 제출 방지

## 관련 기능 ID

- `F-NOTE-01`, `F-NOTE-08`

## 비고

- 좋아요·스크랩·필터(`F-NOTE-03` ~ `F-NOTE-12`)는 본 Flow 완성 후 백로그.
