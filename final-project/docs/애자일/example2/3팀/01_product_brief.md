# Product Brief — 3팀 Tasty MVP (중간 프로젝트 명세 기반)

## 1. 프로젝트명

- Tasty MVP

## 2. 프로젝트 한 줄 소개

- 맛집·식당 피드를 공유하고, 지도에서 식당을 탐색하는 소셜형 MVP

## 3. 문제 정의

- 방문한 식당과 후기를 빠르게 남기고 다른 사람의 피드를 보고 싶다.
- 지도 맥락에서 식당 정보를 찾고 싶다.

## 4. 타깃 사용자

- 맛집 탐방·기록을 즐기는 사용자
- 지역·거리 기준으로 식당을 찾아보고 싶은 사용자

## 5. 핵심 목표

- **Flow A (FLOW-FEED)**: 로그인 후 피드 목록·작성·상세·댓글까지 한 사용자 여정을 완주한다.
- **Flow B (FLOW-MAP-EAT)**: GPS 기반으로 식당 데이터를 불러와 목록·상세를 확인한다. (명세 통합 표의 Map 항목 기준)

## 6. 핵심 사용자 시나리오

1. 로그인한다.
2. 홈 피드에서 게시글 목록을 본다.
3. 글을 작성(제목·본문·명세 범위 내 필드)하고 등록한다.
4. 게시글 상세에서 댓글을 남긴다.
5. 지도에서 위치를 잡고 식당 목록·상세를 본다.

## 7. MVP 범위

### Must (Flow A/B 중심)

- 로그인 `F-Tasty-02`, (선택) 회원가입 `F-Tasty-01`, 로그아웃 `F-Tasty-03`
- 피드: 목록·작성·상세 `F-Tasty-04`, `F-Tasty-05`, `F-Tasty-07`, `F-Tasty-08`
- 댓글 `F-Tasty-09`
- 지도: GPS·식당 불러오기·목록·상세 `F-Tasty-15` ~ `F-Tasty-18`

### Should

- 정렬·필터 `F-Tasty-12`, `F-Tasty-13`
- 사진·별점·식당 검색(작성) `F-Tasty-06`, `F-Tasty-11`, `F-Tasty-14`
- 지도 검색·탐색 고도화 `F-Tasty-19`, `F-Tasty-20`

### Won’t (초기 우선순위 관점)

- 6주 **전체** 배치는 [`04_six_week_iteration.md`](04_six_week_iteration.md)를 따른다(테이스티·마이·공통은 Week5~6에 집중).
- 스프린트 중간에 스코프를 늘릴 때는 Decision Log에 남긴다.

## 8. 차별점

- 피드 + 지도를 같은 앱에서 연결(명세 MVP 메모)

## 9. 제약사항

- 6주·핵심 Flow 2개 (`01_애자일_팀프로젝트_가이드.md`)
- Map 상세는 통합 표가 기준(별도 `##` 절 없음)

## 10. 성공 기준

- 2~5분 내 Flow A·B 데모 성공
- 로딩·에러·빈 상태 포함

## 11. 6주 일정(통합 표 전체)

- `F-Tasty-01` ~ `F-Tasty-33` 주차 배치: [`04_six_week_iteration.md`](04_six_week_iteration.md)
- Issue 주차는 [`03_backlog.md`](03_backlog.md) **주차(Week)** 와 일치. Week6은 테이스티·공통·Freeze에 가깝게 배치됨.

## 12. 팀 선택 사항 (명시)

- Flow B 대안으로 **테이스티 리스트 중심**으로 재편할 경우, `04_six_week_iteration.md`의 Week4~5 지도 묶음과 Week6 테이스티 묶음 순서를 팀에서 조정한다.
