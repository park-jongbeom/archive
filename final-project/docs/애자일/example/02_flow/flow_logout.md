# User Flow — 로그아웃 (FLOW-LOGOUT)

## 목표
- 사용자가 로그아웃하고, 로그인 화면(또는 온보딩)으로 이동한다.

## 시작 조건
- 사용자는 로그인된 상태다.
- 설정/마이페이지 등에서 로그아웃을 수행할 수 있다.

## 주요 단계
1. 사용자가 “로그아웃”을 탭한다.
2. 시스템은 로그아웃을 처리한다(Loading, 필요 시).
3. 성공 시 로그인 상태가 제거되고 로그인(또는 온보딩) 화면으로 이동한다(Success).
4. 실패 시 에러 메시지와 재시도 경로를 제공한다(Error + Retry, 필요 시).

## 종료 조건
- 성공: 로그인(또는 온보딩) 화면으로 이동
- 실패: 현재 화면에서 재시도 가능

## 예외/상태(최소)
- 네트워크 오류/서버 오류(서버 로그아웃이 필요한 경우)
- 로컬 세션 삭제 실패(드문 케이스)
- 중복 탭(연타)

## 필요한 화면
- 설정/마이페이지 화면
- 로그인(또는 온보딩) 화면
