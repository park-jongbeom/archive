# Product Brief (샘플) — “오늘의 기록” (가상)

## 1. 프로젝트명
- 오늘의 기록

## 2. 프로젝트 한 줄 소개
- 하루의 작은 기록을 남기고, 최근 기록을 목록/상세로 확인하는 안드로이드 앱

## 3. 문제 정의
- 기록을 남기고 싶지만 앱을 열고 쓰는 과정이 번거롭다.
- “오늘 무엇을 했는지”를 나중에 다시 찾기 어렵다.

## 4. 타깃 사용자
- 짧은 메모/기록을 남기고 싶은 사용자
- 하루 단위 회고/습관을 기록하는 사용자

## 5. 핵심 목표
- 사용자가 10초 안에 기록을 남기고, 다시 확인할 수 있게 한다.

## 6. 핵심 사용자 시나리오
1. 사용자가 로그인한다.
2. 홈에서 기록 목록을 본다.
3. 기록을 선택해 상세를 확인한다.
4. (선택) 설정에서 탈퇴한다.

## 7. MVP 범위
### Must
- 로그인(성공/실패/로딩)
- 홈 목록(로딩/빈/에러/재시도)
- 상세(로딩/에러/빈/재시도)

### Should
- 검색(간단)
- 목록 정렬(최신순 고정)

### Won’t (이번 범위 아님)
- SNS 공유
- 복잡한 통계/리포트

## 8. 차별점
- “완성도 높은 데모(Flow 2개)”를 우선으로 설계된 단순한 경험

## 9. 제약사항
- 6주 내 구현 범위 제한(Flow 2개 중심)
- 서버 구현은 최소 또는 BaaS 사용

## 10. 성공 기준
- 2~5분 내 데모를 실패 없이 수행한다.
- 로딩/에러/빈 상태를 최소 1회씩 보여준다.
