# Weekly Status Issue 작성 안내 (학생 / 팀 PO용)

> 매주 1번 GitHub에 진행 상황을 정해진 양식으로 적습니다. **5분이면 끝**입니다.
> 강사가 자동으로 파싱해서 팀별 신호등을 만들고, 1:1 때 어디 짚을지 판단합니다.

---

## 1. Issue Template 1회 셋업 (팀 PO가 1주차에 한 번)

### 1단계 — 템플릿 파일 다운로드
강사가 전달하는 [`../templates/weekly-status.md`](../templates/weekly-status.md) 내용을 복사.

### 2단계 — 팀 레포에 추가
팀 레포의 다음 경로에 파일 생성 + 커밋:
```
.github/
└── ISSUE_TEMPLATE/
    └── weekly-status.md   ← 복사한 내용 그대로 붙여넣기
```

### 3단계 — 라벨 생성
팀 레포 → Issues → Labels → **New label**
- 이름: `weekly-status`
- 색상: 자유 (예: 파란색)

### 4단계 — Issue Template 작동 확인
팀 레포 → Issues → New issue → **"Weekly Status"** 항목이 보이면 OK.

---

## 2. 매주 작성 (PO가 일요일 23:59까지)

### 절차

1. 팀 레포 → Issues → **New issue** → Weekly Status 선택
2. 제목 자동 입력됨: `[Status] Week N - 팀명` → **N과 팀명을 실제 값으로 교체**
   - 예: `[Status] Week 2 - 1팀`
3. 본문 6개 필드 작성 (5분)
4. 라벨 `weekly-status` 자동 적용 확인
5. **Submit new issue**

### 본문 필드 작성 가이드

#### Brief 상태
- 작성 완료 → `- [x] Brief 작성 완료`
- 미작성 → `- [ ] Brief 작성 완료`

#### MVP 분포
가이드 § 4-4 기준. 현재 시점의 Must / Should / Won't 항목 **개수**.
```
- Must: 4
- Should: 2
- Won't: 3
```
빈칸(`__`)으로 두지 말 것. 모르겠으면 0이라도 쓸 것.

#### Flow
가이드 § 5-2-1 기준. **확정된** Flow만.
```
- 확정 Flow 수: 2
- Flow 이름:
  - FLOW-LOGIN
  - FLOW-RECORD-CREATE
```

#### Decision Log
이번 주 (지난 일주일간) 결정 사항 개수와 가장 큰 결정 한 줄.
```
- 이번 주 결정 수: 1
- 가장 큰 결정 한 줄: 워치 양방향 동기화는 Won't로 이동
```

#### Demo Scenario
- 이번 주 데모 시나리오 노션에 작성됨 → `- [x]`
- 데모 한 줄: 발표에서 보여줄 핵심 한 문장

#### 이번 주 한 줄
이번 주에 진짜로 굴러간 것 1개. 추상적 표현 금지.
- ❌ "프로젝트 진행"
- ✅ "로그인 성공 경로 데모 가능"

---

## 3. 절대 하지 말 것

- ❌ **숫자를 부풀리기** — 강사가 1:1에서 spot-check 합니다. 거짓 보고는 평가가 아니라 멘토링 신뢰를 깹니다
- ❌ 빈칸(`__`) 그대로 제출 — 모르면 `0` 또는 `?` 라도 쓰고 막힌 점에 적기
- ❌ 매주 새 라벨 만들기 — `weekly-status` 라벨 1개만 유지
- ❌ 한 주에 Status Issue 2개 작성 — 1주 = 1Issue. 잘못 썼으면 Edit
- ❌ 늦게 작성 후 날짜 조작 — GitHub 작성 시간이 자동 기록됨

---

## 4. 자주 묻는 질문

### Q. Brief 미작성인데 작성 완료라고 체크하면?
강사가 매주 노션 PDF로 직접 확인합니다. 불일치 시 1:1에서 짚입니다. 부풀리지 마세요.

### Q. Won't 항목이 정말 0개면?
**그게 🔴 신호입니다.** 가이드 § 4-4 — *"Won't가 없으면 범위가 무한 확장"*. 0개를 적는 게 정직하면, 다음 1:1 때 Won't 추가 회의를 하면 됩니다.

### Q. Flow가 5개면 줄여서 적나?
**아니요. 5라고 쓰세요.** 강사가 *"5개는 1~2개로 좁혀야"* 신호로 읽고 1:1에서 줄이도록 코칭합니다.

### Q. PO가 휴가/병가면?
부 PO 또는 팀원 누구든 작성 가능. 단 라벨/제목 형식만 맞추세요.

### Q. Week N 표기 기준?
부트캠프 일정 기준 주차. Week 1 = 5월 4~10일 (예시), Week 2 = 5월 11~17일 등. 강사가 OT에서 공지.

### Q. Issue를 닫아도 되나요?
**아니요. 6주 동안 열어두세요.** 다음 주 작성하면 자동으로 이전 주차 Status는 비활성화됩니다 (라벨 + 제목 Week N으로 강사가 구분).
